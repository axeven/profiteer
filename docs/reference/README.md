# Reference Documentation

This directory contains **reference materials and supporting documentation** for the Profiteer app.

## Purpose

Reference documentation provides:
- Troubleshooting guides
- Debugging information
- Historical records
- Supporting materials that don't fit elsewhere

## Directory Contents

- **credential-debugging.md** - Firebase credential troubleshooting guide

## When to Use Reference Docs

Reference documentation is for:
- **Troubleshooting**: Step-by-step debugging guides
- **Historical Context**: Background information on decisions
- **Supporting Materials**: Documentation that supports specifications or guidelines
- **Quick Reference**: Lookup tables, cheat sheets

## Reference vs Other Documentation

| Specifications | Guidelines | Reference |
|---------------|-----------|-----------|
| WHAT to build | HOW to build | Supporting info |
| Requirements | Patterns | Troubleshooting |
| Authoritative | Instructional | Supplementary |

## Finding What You Need

- **Need to understand a feature?** → Check `docs/specs/`
- **Need to implement something?** → Check `docs/guides/`
- **Having an issue?** → Check `docs/reference/`
- **Need historical context?** → Check `CHANGELOG.md` or `docs/archived/`

## Adding Reference Documentation

Reference docs should:
1. Support existing specifications or guidelines
2. Provide practical troubleshooting help
3. Include clear examples
4. Be kept up-to-date
5. Not duplicate specifications or guidelines

## Examples of Good Reference Docs

- Troubleshooting guides
- Configuration references
- Error message catalogs
- Debugging checklists
- Environment setup guides
